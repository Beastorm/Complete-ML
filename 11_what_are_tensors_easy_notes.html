What is a Tensor?

A tensor is a container for numbers. In rare cases it can store other things, but in ML/DL it is almost always numbers.

A tensor is also a general term for scalars, vectors, matrices, and higher-dimensional arrays.

Scalar0D tensor
→
Vector1D tensor
→
Matrix2D tensor
→
Higher Tensor3D, 4D, 5D...
In programming, especially NumPy, a tensor is usually represented as an array/ndarray. In this context, NumPy arrays and tensors can be thought of as the same practical object.
0️⃣

0D Tensor: Scalar

A 0D tensor is a single number. It is also called a scalar.

Examples: 4, 12, 20000

import numpy as np x = np.array(4) print(x) # 4 print(x.ndim) # 0
  • ✓It has no axis.
  • ✓Its rank/dimension is 0.
  • ✓Its size is 1 because it stores one item.
1️⃣

1D Tensor: Vector

A 1D tensor is a collection/list of scalar values. It is also called a vector.

1
2
3
4
v = np.array([1, 2, 3, 4]) print(v.ndim) # 1 print(v.shape) # (4,) print(v.size) # 4
Important confusion: a vector can have 4 values, so we may say it is a “4-dimensional vector.” But as a tensor, it is still a 1D tensor because it has one axis.
📏

Tensor dimension

Number of axes. Here it is 1.

🧮

Vector dimension

Number of values/components. Here it is 4.

2D Tensor: Matrix

A 2D tensor is a collection of vectors. It is also called a matrix.

A matrix has two axes: rows and columns.

  • ✓Axis 2 can be understood as columns.
  • ✓Shape tells how many items exist along each axis.

Rank, Axis, Shape, and Size

These terms are very important when working with tensors.

↔️

Axis

A direction in the tensor. A matrix has row and column axes.

🏷️

Rank

Number of axes. Same as tensor dimension in this context.

📦

Shape

Number of items along each axis.

#️⃣

Size

Total number of items = product of shape values.

Tensor Rank / ndim Shape Size Meaning
Scalar 4 0 () 1 Single number.
[1, 2, 3] 1 (3,) 3 Vector with three components.
3×3 matrix 2 (3, 3) 9 3 rows and 3 columns.
3 sentences × 2 words × 4 vocab 3 (3, 2, 4) 24 NLP text tensor example.
Formula: size = multiply all numbers in the shape. Example: shape (3, 2, 4) has 3 × 2 × 4 = 24 items.

Practical ML Examples of Tensors

The second half of the video gives practical examples from ML/DL where you will see 1D to 5D tensors.

1️⃣

Example of 1D Tensor: One Student

Suppose we have student data with input columns like CGPA, IQ, and State. The target is whether placement happened or not.

After encoding the state column, one student can be represented as a vector:

student = [8.8, 90, 0] # 8.8 = CGPA # 90 = IQ # 0 = encoded state, e.g. West Bengal
  • ✓This is a 1D tensor because it has one axis.
  • ✓It is also a 3-dimensional vector because it has 3 features/components.
  • ✓If there are 50 input columns, one student is still a 1D tensor, but the vector has 50 components.
This is the key distinction from the video: tensor dimension and vector dimension are not always the same thing.
2️⃣

Example of 2D Tensor: Tabular ML Dataset

If one student is one vector, then many students together form a matrix.

For example, 1000 students with 3 input features create an input matrix:

X shape = (1000, 3) → 1000 rows/students and 3 columns/features.
X = [ [8.8, 90, 0], [7.2, 102, 1], ... ] # X is a 2D tensor / matrix # y = placement labels is a 1D tensor of length 1000
  • ✓Input data for tabular ML is commonly stored as a 2D tensor/matrix.
  • ✓The target/output column is often stored separately as a 1D tensor/vector.
  • ✓This is one of the most common tensor structures in basic Machine Learning.
3️⃣

Example of 3D Tensor: NLP Text Data

Machine Learning algorithms do not understand raw text directly. Text must be converted into numbers/vectors. This process is called vectorization.

Simple one-hot example

Suppose we have these sentences:

hi nitish
hi rahul
hi ankit

Vocabulary: hi, nitish, rahul, ankit

hi = [1, 0, 0, 0] nitish = [0, 1, 0, 0] rahul = [0, 0, 1, 0] ankit = [0, 0, 0, 1] sentence_1 = [hi, nitish] # shape (2, 4) sentence_2 = [hi, rahul] # shape (2, 4) sentence_3 = [hi, ankit] # shape (2, 4) text_tensor = [sentence_1, sentence_2, sentence_3] # shape = (3, 2, 4)
  • ✓Each word becomes a vector.
  • ✓Each sentence becomes a 2D tensor/matrix: words × vector-size.
  • ✓A collection of sentences becomes a 3D tensor.
In real NLP, there are many vectorization techniques. The video uses a simple method to build intuition.

Example of 4D Tensor: Images

An image is a collection of pixels. Each pixel has numeric values. For color images, values are often stored in three channels: Red, Green, Blue.

Single Color Imageheight × width × channels
→
3D Tensorexample (480, 720, 3)
→
Batch of Images4D tensor

If you have 60 color images of size 480 × 720, the batch shape becomes:

Batch shape = (60, 480, 720, 3)
  • ✓Single color image = 3D tensor.
  • ✓Multiple color images = 4D tensor.
  • ✓Deep learning models for computer vision commonly work with 4D image batches.
🎬

Example of 5D Tensor: Videos

A video is basically a fast sequence of images/frames. If images pass in front of our eyes very quickly, we perceive motion.

The video explains that 30 FPS, 60 FPS, 120 FPS, etc. mean how many frames/images are shown per second.

Frameone image
→
Videocollection of frames
→
Video Batchcollection of videos
→
5D Tensorvideos × frames × H × W × C

Example calculation from the video

Suppose we have:

4 videos
60 seconds each
30 FPS
480 × 720 resolution
3 RGB channels

Frames per video = 60 × 30 = 1800 frames

Tensor shape = (4, 1800, 480, 720, 3)

Total values = 4 × 1800 × 480 × 720 × 3 = 7,464,960,000 numeric values.

If each value takes 32 bits, this becomes roughly 27 GB of raw storage.

  • ✓This shows why video ML/DL is computationally heavy.
  • ✓Formats like MP4, MPEG, and MKV reduce storage using compression.
  • ✓For ML processing, video data is often represented as high-dimensional tensors.

Final Super Simple Summary

  • Tensor: a container/data structure for numbers.
  • 0D tensor: scalar, a single number.
  • 1D tensor: vector, a list of numbers.
  • 2D tensor: matrix, a table of numbers.
  • 3D tensor: collection of matrices; common in NLP and time-series data.
  • 4D tensor: batch of images; common in computer vision.
  • 5D tensor: batch of videos; used in video processing.
  • Rank / ndim: number of axes in a tensor.
  • Shape: number of items along each axis.
  • Size: total number of items, calculated by multiplying shape values.
  • Important distinction: a vector may have many components, but as a tensor it can still be 1D.
  • Main lesson: ML/DL data is stored and processed as tensors, so tensors are a core practical concept.

Data Engineer

Builds data infrastructure and pipelines.

📊

Data Analyst

🧠

Why Do These Different Roles Exist?

The answer comes from the previous video on Machine Learning Development Life Cycle.

To build an ML product, a team goes through many stages: planning, data collection, preprocessing, EDA, modeling, evaluation, deployment, and optimization. In a big company, one person usually cannot do everything.

Data WorkCollect, store, pipeline
→
AnalysisClean, analyze, report
→
ModelingTrain and select model
→
ProductionDeploy, monitor, optimize
Different job roles are created because different stages of the ML lifecycle need different skills.
🛠️

1Data Engineer

Simple meaning: A Data Engineer brings the data and prepares the data infrastructure so that analysts and scientists can use it.

The teacher says the Data Engineer’s job is to “bring the data to the table.”

OLTP vs OLAP / Data Warehouse Example

In a company like Flipkart, the main website/app runs on a live operational database. This type of database is used for live user transactions and is similar to OLTP — Online Transaction Processing.

The analytics or data science team should not directly work on the live production database because it can affect the running website/app. So a separate analytical system is created, often called OLAP — Online Analytical Processing or a Data Warehouse.

Live DatabaseOLTP / production data
→
Data EngineerExtract, transform, load
→
Data WarehouseOLAP / analytics data
→
Analytics TeamUse safely for analysis/ML

Main responsibilities

  • ✓Create data warehouses from operational databases.
  • ✓Collect data from multiple sources: company databases, APIs, third-party sources, web scraping, etc.
  • ✓Move and store data in suitable databases or warehouses.
  • ✓Build data pipelines and APIs so other teams can access data easily.
  • ✓Maintain databases, warehouses, and pipelines over time.
  • ✓Monitor whether pipelines are working correctly and data is reaching on time.

Skills needed

Strong software engineering
Data structures & algorithms
Programming: Java / Scala / Python
SQL databases
NoSQL databases
Big data tools: Spark / Hadoop / Hive
Cloud: AWS / GCP
Distributed systems
System design
Pipeline tools like Airflow
Data Engineering is highly important in big organizations because “data is gold.” It can pay well because the skill set is difficult and relatively scarce.

2Data Analyst

After the Data Engineer brings the data, the Data Analyst cleans, analyzes, visualizes, and explains what is happening in the data.

Simple meaning: A Data Analyst reads the data and tells its story.

DataFrom data engineer
→
AnalysisFind what happened and why
→
ReportsExplain insights to business

What questions does a Data Analyst answer?

  • ✓Why did a product fail?
  • ✓Why did profit decrease last year?
  • ✓What happened in the past data?
  • ✓What patterns or insights can management use?

Main responsibilities

  • ✓Clean and organize raw data.
  • ✓Analyze data to find insights.
  • ✓Create data visualizations, charts, and dashboards.
  • ✓Create and maintain reports and slides for management.
  • ✓Collaborate with colleagues and management to explain what the data says.
  • ✓Help improve data collection procedures and reporting processes.

Data Analyst vs Business Analyst

📈

Data Analyst

More data/tool/engineering-oriented. Uses SQL, Python/R, visualization tools, reports, and dashboards.

💼

Business Analyst

More management/business-oriented. Uses business understanding, Excel, reports, and communication heavily.

Skills needed

Statistics
Python or R
Analytical thinking / common sense
Domain knowledge
Communication skills
Data mining
Data visualization
Data storytelling
SQL
Excel / BI tools
Data storytelling is very important. A good Data Analyst does not just show charts; they present an interesting, useful story from data.

3Data Scientist

The video gives a famous description:

A Data Scientist is someone who is better at statistics than a software engineer and better at software engineering than a statistician.

A Data Scientist is like a full-stack person in the data field. They should understand the full Machine Learning Development Life Cycle, even if they do not do every step in a large company.

Data Analyst vs Data Scientist

🕰️

Data Analyst looks at the past

What happened? Why did it happen? What does the data say about past performance?

🔮

Data Scientist builds for the future

Uses data understanding to build models, predictions, recommendation systems, optimization systems, etc.

Examples of Data Scientist work

  • ✓Build a recommendation engine to increase sales.
  • ✓Create a model that improves customer experience.
  • ✓Build prediction systems based on data.
  • ✓Optimize routes or business processes.
  • ✓Create best-performing ML models using reports and data.

Role in different company sizes

🏢

Large Company

Data Engineer brings data, Data Analyst creates reports, Data Scientist focuses mainly on creating the best model.

🏪

Small Company

The Data Scientist may have to perform multiple roles: data gathering, analysis, modeling, and sometimes deployment support.

Skills needed

Statistics
Machine Learning
Mathematics
Programming
Data handling
Model evaluation
Business/domain understanding
Communication
Some software knowledge
If you want to enter the data science field, the video suggests Data Scientist is often the role many learners target, but it requires broad and deep preparation.

4Machine Learning Engineer

Many learners think that because they are learning Machine Learning, they must become an ML Engineer. The video clarifies that an ML Engineer has a specific production-focused role.

Simple meaning: An ML Engineer takes a trained model and makes it work reliably inside a real product.

Data ScientistCreates model
→
ML EngineerProduction bridge
→
Software ProductWebsite / app / backend

Why is this role needed?

Data Scientists may not know software development deeply. Software developers may not know data science deeply. To integrate a trained model into a website, app, or production system, someone needs to understand both sides. That person is the ML Engineer.

Main responsibilities

  • ✓Deploy Machine Learning models into production.
  • ✓Make models production-ready and scalable.
  • ✓Optimize model performance in real systems.
  • ✓Create retraining scripts and manage how new data comes for retraining.
  • ✓Take backups and handle model versioning/maintenance style work.
  • ✓Monitor and maintain deployed models.

Skills needed

Programming
Distributed systems
Model deployment
ML model understanding
Software engineering concepts
System design
Production monitoring
Cloud / backend basics
ML Engineer is the bridge between the data science world and the software engineering/product world.
🗺️

How These Roles Map to the ML Lifecycle

MLDLC Stage Primary Role What They Do
Planning / understanding problem Data Scientist + Business Team Understand the problem, business goal, and possible ML solution.
Data gathering Data Engineer Bring data from databases, APIs, scraping, warehouses, and pipelines.
Preprocessing / cleaning Data Analyst + Data Scientist Clean, organize, and prepare data for analysis/modeling.
EDA and reporting Data Analyst Analyze data, create visualizations, reports, dashboards, and insights.
Modeling Data Scientist Train, evaluate, tune, and select ML models.
Deployment ML Engineer Put the trained model into a real software product or production system.
Monitoring and optimization ML Engineer + Data Scientist Monitor model behavior, maintain it, retrain, optimize, and scale.

Quick Comparison

Role Core Job Mostly Looks At Main Output Best Fit For
Data Engineer Bring, store, and pipeline data. Data infrastructure and systems. Data warehouse, data pipelines, clean access to data. People who like hardcore software, databases, systems, big data.
Data Analyst Analyze data and explain insights. Past: what happened and why. Reports, dashboards, visualizations, business insights. People who like data, charts, business questions, storytelling.
Data Scientist Build models and future-facing solutions. Future: what can happen and what can be built. ML models, predictions, recommendation systems, optimization systems. People who like ML, statistics, math, programming, and problem solving.
ML Engineer Deploy and maintain ML models in production. Production systems and scalability. APIs, deployed models, monitoring, retraining pipelines. People who like software engineering + ML deployment.

Which Role Should You Choose?

The video suggests that after understanding the roles, you should explore real job listings and see what companies are asking for.

🧱

Choose Data Engineer if...

You like software, databases, pipelines, cloud, distributed systems, and big data.

📊

Choose Data Analyst if...

You like exploring data, dashboards, business insights, reports, and storytelling.

🧠

Choose Data Scientist if...

You like statistics, math, ML algorithms, modeling, and solving business problems using data.

🚀

Choose ML Engineer if...

You like software engineering and want to deploy, scale, monitor, and maintain ML models.

Practical advice from the video: Do not only ask seniors or teachers what to learn. Go to job websites, read real job descriptions, and build the skills companies are actually asking for.

Final Super Simple Summary

  • Different job roles exist because one person cannot handle the entire ML lifecycle in large companies.
  • Data Engineer: builds data pipelines, warehouses, APIs, and handles databases/big-data systems.
  • Data Analyst: cleans/analyzes data, creates reports, visualizations, dashboards, and tells the story of past data.
  • Data Scientist: uses statistics, ML, programming, and domain understanding to build models and future-facing data products.
  • ML Engineer: deploys trained models into production and handles scaling, monitoring, retraining, and maintenance.
  • Data Analyst vs Data Scientist: Analyst explains what happened in the past; Scientist builds models for future predictions/solutions.
  • Data Engineer vs ML Engineer: Data Engineer prepares data infrastructure; ML Engineer prepares model infrastructure.
  • Career advice: Check real job descriptions and develop the skills companies ask for.

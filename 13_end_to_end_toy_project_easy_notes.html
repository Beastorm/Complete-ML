Loading and Checking Data

The video uses Google Colab and a small uploaded file called placement.csv.

import numpy as np import pandas as pd df = pd.read_csv('placement.csv') df.head() df.shape # around 100 rows and 4 columns
  • ✓One extra unnamed/index-like column is unnecessary.
  • ✓df.info() shows there are no missing values in this toy dataset.
🧹

Preprocessing

In this toy dataset, preprocessing is simple because there are no missing values. The main task is removing the unnecessary first column.

df = df.iloc[:, 1:] # keep only cgpa, iq, placement
In real projects preprocessing can include missing values, outliers, duplicate rows, wrong formats, encoding, scaling, and more.
📊

EDA and Scatter Plot

EDA shows how the data looks and whether classes can be separated. A scatter plot between CGPA and IQ is drawn and colored by placement.

import matplotlib.pyplot as plt plt.scatter(df['cgpa'], df['iq'], c=df['placement'])
  • ✓One color shows students placed; another shows not placed.
  • ✓The pattern looks separable by a line.
  • ✓Because of this, Logistic Regression is a suitable simple model.
🧩

Feature Selection and X/y

For this toy project, both input columns are used: CGPA and IQ. The placement column is the output.

X = df.iloc[:, 0:2] y = df.iloc[:, -1] # X = independent variables / inputs # y = dependent variable / target
CGPA and IQ are independent variables. Placement depends on them, so it is the dependent variable.
⚖️

Train-Test Split and Scaling

The model should be tested on data it has not seen during training. So the dataset is split into training and testing parts.

from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.1) from sklearn.preprocessing import StandardScaler scaler = StandardScaler() X_train = scaler.fit_transform(X_train) X_test = scaler.transform(X_test)
  • ✓With test_size=0.1, about 10% data is hidden for testing.
  • ✓Scaler is fitted only on training data, then applied to test data.
  • ✓Scaling helps when feature ranges differ and is especially important for distance-based models.
🤖

Training Logistic Regression

Logistic Regression learns a line/decision boundary that separates placed and not-placed students.

from sklearn.linear_model import LogisticRegression clf = LogisticRegression() clf.fit(X_train, y_train)

Training may look simple and quick for small datasets, but the same idea extends to larger models and datasets.

📏

Evaluation and Accuracy

The model predicts on the hidden test data and the predictions are compared with actual answers.

y_pred = clf.predict(X_test) from sklearn.metrics import accuracy_score accuracy_score(y_test, y_pred) # example: 0.9
  • ✓If 9 out of 10 predictions are correct, accuracy is 90%.
  • ✓The video also shows decision boundary visualization using mlxtend.
  • ✓Misclassified points show where the model made mistakes.
📦

Exporting and Deployment Idea

After the model looks acceptable, it is saved using pickle and then can be used inside a website/app.

import pickle pickle.dump(clf, open('model.pkl', 'wb'))
  • ✓Pickle converts the trained model object into a file.
  • ✓A website can take IQ and CGPA as input and return placement prediction.
  • ✓Cloud deployment options mentioned: Heroku, AWS, and Google Cloud Platform.

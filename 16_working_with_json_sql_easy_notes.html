What is JSON?

JSON means JavaScript Object Notation. It is a universal, lightweight format that most programming languages can read.

  • ✓APIs commonly return data in JSON format.
  • ✓Python, Java, JavaScript, and many other languages can parse JSON.
  • ✓JSON is common on Kaggle and API-based data projects.
🐼

Reading JSON with pandas

The video uses a recipe/cuisine dataset where each record contains dish id, cuisine type, and ingredients.

import pandas as pd df = pd.read_json('train.json') df.head()

read_json works very similarly to read_csv.

  • ✓Pass a local JSON file path.
  • ✓You can also pass a URL directly.
  • ✓Useful parameters include encoding, dtype, convert_dates, nrows/chunks depending on data.
url = 'https://example.com/data.json' df = pd.read_json(url)
🗄️

What is SQL?

SQL means Structured Query Language. It is used to work with relational databases and fetch/manipulate structured data.

🔍

Fetch

Select rows and columns.

🧱

Manage

Create and modify tables/databases.

🎯

Filter

Use WHERE, conditions, and joins.

🧰

Setting up SQL Data with XAMPP

The video uses a world.sql dataset from Kaggle and loads it into MySQL using XAMPP.

  1. ✓Download and install XAMPP.
  2. ✓Open XAMPP Control Panel.
  3. ✓Start Apache web server and MySQL database server.
  4. ✓Open localhost/phpmyadmin.
  5. ✓Create a database named world.
  6. ✓Import world.sql.
After import, tables such as city, country, and countrylanguage become available.
🔗

Connecting Python to MySQL

Python needs a connector library to talk to MySQL.

!pip install mysql.connector import mysql.connector conn = mysql.connector.connect( host='localhost', user='root', password='', database='world' )
  • ✓host='localhost' means MySQL is running on your own machine.
  • ✓Default XAMPP username is usually root.
  • ✓Password may be empty in default local setup.
  • ✓The connection object is passed to pandas.
🐼

Reading SQL Query Results into pandas

Use pd.read_sql_query() with an SQL query and the connection object.

df = pd.read_sql_query('SELECT * FROM city', conn) df_india = pd.read_sql_query("SELECT * FROM city WHERE CountryCode='IND'", conn) df_country = pd.read_sql_query('SELECT * FROM country WHERE LifeExpectancy > 60', conn)
  • ✓You can query any table such as city, country, or countrylanguage.
  • ✓Use SQL WHERE conditions to filter before loading into pandas.
  • ✓The output becomes a normal DataFrame for analysis/ML.
⚙️

Useful read_sql_query Options

OptionUse
sqlThe SQL query string, such as SELECT * FROM city.
conThe database connection object.
index_colUse a column as DataFrame index.
parse_datesConvert date columns into datetime.
chunksizeLoad large query results in chunks to reduce RAM pressure.
If you know SQL, you can filter/manipulate data before it even enters pandas.

Final Super Simple Summary

  • JSON is a universal text-based format commonly returned by APIs.
  • pandas read_json can load local JSON files or JSON from a URL.
  • SQL is used to query relational databases.
  • XAMPP can run Apache and MySQL locally; phpMyAdmin helps import SQL files.
  • The world.sql dataset creates tables such as city, country, and countrylanguage.
  • mysql.connector acts as a bridge between Python and MySQL.
  • pd.read_sql_query converts SQL query output into a pandas DataFrame.
  • SQL filtering can reduce data before it reaches pandas.

Earlier topics explained the what and why of Machine Learning. This video starts answering the how: how a real Machine Learning product is built.

The teacher compares MLDLC with SDLC — Software Development Life Cycle. Just like software products follow a structured process, ML-based products also need a structured process.

2Gathering the Data

Without data, Machine Learning is not possible. In college projects, data may come easily from Kaggle, CSV files, or teachers. But in real companies, data is often specific and not easily available.

📄

CSV / Files

Ready-made files are easiest to use.

🔌

APIs

Fetch JSON/data using Python and convert it to required format.

🕸️

Web Scraping

Collect data from websites when no direct file/API exists.

🗄️

Databases

Use database, data warehouse, ETL, or big-data systems.

Important data sources/details

  • ✓APIs: You may hit an API with Python, get data in JSON, and convert it into CSV or another useful format.
  • ✓Web scraping: Travel or price-comparison websites often collect data from multiple web pages.
  • ✓Data warehouse: Instead of running ML directly on a live production database, companies may create a warehouse using ETL — Extract, Transform, Load.
  • ✓Big data: If data is huge, it may be handled through clusters or big-data tools before ML work begins.
Goal of this step: bring data from the right sources and store it in a form where further ML work can happen safely.
🧹

3Data Preprocessing

Raw data is usually dirty. It may have missing values, outliers, duplicate rows, noisy entries, incompatible formats, and scale differences.

Preprocessing means making changes before training so the ML algorithm can consume the data properly.

👥

Duplicates

Remove duplicate records.

❓

Missing Values

Remove or fill missing data.

🚨

Outliers

Detect and handle extreme values.

📏

Scaling

Bring columns to comparable scale.

Example: if one column has values in crores and another has decimal values, distance-based algorithms may get confused. Scaling helps bring values into a suitable range.
🔍

4Exploratory Data Analysis — EDA

EDA means analyzing and understanding the data before building the model. You study relationships between input and output and discover hidden patterns.

1️⃣

Univariate Analysis

Study one column at a time: distribution, frequency, spread, etc.

2️⃣

Bivariate Analysis

Study relationship between two columns.

3️⃣

Multivariate Analysis

Study relationship among three or more columns together.

  • ✓Create graphs and visualizations to understand the data.
  • ✓Study input-output relationships.
  • ✓Look for class imbalance, such as too many cat images and too few dog images.
  • ✓Build a concrete understanding of the dataset before making modeling decisions.
Analogy from the video: If you have 6 hours to cut a tree, spend 4 hours sharpening the axe. Similarly, spend enough time understanding your data before modeling.
🧩

5Feature Engineering and Feature Selection

Features are input columns. They are extremely important because the output depends on them.

🛠️

Feature Engineering

Create new features or intelligently transform existing features so the model can learn more easily.

Example: in a house price model, multiple house-related columns can be combined or transformed into a more useful feature like area/size.

✂️

Feature Selection

Select only important features and remove useless or less helpful columns.

Reasons: not every input affects the output, and too many columns increase training time.

Feature work is very crucial. The video says future lessons will spend multiple videos on feature engineering and feature selection.
🤖

6Model Training, Evaluation, and Selection

After cleaning the data and creating good features, you are ready to train models. Usually, you do not train only one algorithm. You try multiple algorithms and compare them.

Why train multiple algorithms?

No single algorithm is best for every dataset. One algorithm may perform well on text data, while another may perform better on a different type of data. So you try many options and decide based on results.

📏

Evaluation Metrics

Metrics tell how well a model is performing.

🎚️

Hyperparameter Tuning

Adjust model settings, like changing TV settings for best picture/audio.

🧠

Ensemble Learning

Combine multiple models to create a stronger model.

  • ✓Classification, regression, and clustering can have different evaluation metrics.
  • ✓Hyperparameters are settings of the model that can be tuned to improve performance.
  • ✓Ensemble techniques combine models using approaches such as bagging, boosting, stacking, or voting.
  • ✓The final result is a trained model that is ready to be converted into a usable product.
🚀

Deployment can use cloud platforms like AWS or GCP. Once deployed, the model becomes online and starts serving user requests.
🧪

8Beta Testing

After deployment, the product should not always be released to all users immediately. First, release it to a small group of trusted or loyal users and collect feedback.

Small ReleaseTrusted users first
→
Collect FeedbackCheck if model works well
→
ImproveFix issues before full release
  • ✓Software updates often roll out to a small user group first.
  • ✓Beta testing helps decide whether the model is working properly in real conditions.
  • !If feedback is bad, go back to earlier stages: data, preprocessing, feature selection, algorithm choice, or tuning.
MLDLC is iterative. If something goes wrong, you may have to revisit previous steps and improve them.
⚙️

9Optimization and Maintenance

If beta testing works well, the model can be released to all users. But before and after full launch, the whole system must be optimized.

💾

Backups

Take backups of the model and data so you can recover if something breaks.

⚖️

Load Balancing

Handle many users by distributing requests properly.

🔁

Retraining

Decide how frequently the model should be retrained.

Model rotting / performance decay

Over time, a model’s performance can become worse because real-world data changes. This is sometimes called model rotting or performance decay.

Example from the video: A mask detection system may work well initially, but new mask types may appear later. If the model has not seen such examples, it can make mistakes. So new data must be collected and the model must be retrained.

Optimization also means automating repeated work and reducing unnecessary cost wherever possible.

MLDLC Practical Checklist

Step Main Question Important Tasks Output of Step
1. Framing What exactly are we building? Define problem, users, cost, team, ML type, learning type, possible data source. Clear project direction.
2. Data Gathering Where will data come from? CSV, API, scraping, databases, warehouse, ETL, big-data tools. Raw data stored safely.
3. Preprocessing Is data clean enough for ML? Handle duplicates, missing values, outliers, formats, scaling. Clean usable data.
4. EDA What is inside the data? Visualize, study relationships, univariate/bivariate/multivariate analysis, imbalance. Understanding and insights.
5. Features Which inputs should the model use? Create new features, transform features, remove useless features. Better feature set.
6. Training Which model performs best? Train models, evaluate metrics, select model, tune hyperparameters, use ensembles. Final trained model.
7. Deployment How will users use it? Save model, create API, connect frontend/backend, deploy on cloud/server. Live ML product.
8. Beta Testing Does it work for real users? Release to limited trusted users, collect feedback, fix issues. Validated product.
9. Optimization How do we keep it reliable? Backups, load balancing, retraining schedule, automation, cost optimization. Stable production system.

Final Super Simple Summary

  • MLDLC is a guideline/process for building an end-to-end ML product.
  • Do not stop at accuracy: real ML work includes deployment, testing, monitoring, and maintenance.
  • Step 1: Frame the problem clearly before starting.
  • Step 2: Gather data from files, APIs, scraping, databases, warehouses, or big-data systems.
  • Step 3: Preprocess data by handling duplicates, missing values, outliers, and scaling.
  • Step 4: Use EDA to deeply understand the data and hidden relationships.
  • Step 5: Engineer and select features to improve learning and reduce training cost.
  • Step 6: Train many models, evaluate them, tune settings, and select the best.
  • Step 7: Deploy the model through an app/API so users can actually use it.
  • Step 8: Beta test with trusted users and collect feedback.
  • Step 9: Optimize, automate, backup, load-balance, monitor, and retrain over time.

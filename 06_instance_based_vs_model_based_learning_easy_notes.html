Instance-Based vs Model-Based Learning

This video explains Machine Learning types based on one question: does the algorithm simply remember examples, or does it learn a general rule/model from them?

🧠
Instance-Based Memorize examples and compare with new data
📐
Model-Based Learn a rule, function, or decision boundary
Basis Human Analogy Instance-Based KNN Example Model-Based Decision Boundary Differences Algorithms Summary

1. What is the Basis of This Classification?

Earlier videos classified Machine Learning based on supervision and production behavior. This video classifies Machine Learning based on:

How does the model learn from data?
📚

Instance-Based Learning

The algorithm stores/memorizes the training examples and uses them later for prediction.

🧩

Model-Based Learning

The algorithm learns an underlying pattern, rule, function, or concept from the data.

👨‍🎓

2. Human Learning Analogy

The teacher explains this using how students learn:

📝

Memorizing

Sometimes students do not understand the concept, so they memorize answers for exams.

This is similar to Instance-Based Learning.
💡

Understanding

Sometimes students understand the underlying concept and can solve new questions.

This is similar to Model-Based Learning.
📌

3. Instance-Based Learning

Instance-Based Learning means the algorithm keeps the training data examples with itself.

When a new data point comes, it compares that new point with stored training points and predicts based on similarity.

Store DataKeep all examples
→
New QueryNew student/data point
→
CompareFind similar points
→
PredictUse nearby examples
Important point: In Instance-Based Learning, there is no heavy “learning” during training. The real work happens when a new point comes.
👥

5. Model-Based Learning

Model-Based Learning means the algorithm tries to learn the underlying pattern or principle from the training data.

Instead of only storing examples, it creates a mathematical model/function that represents the relationship between input and output.

Training DataInput + output
→
Learn PatternFind relationship
→
Create ModelFunction + parameters
→
PredictUse learned rule
Model-Based Learning focuses on the rule behind the data, not just the individual training examples.
➗

7. What is a Model and What are Parameters?

In Model-Based Learning, the algorithm creates a model. A model is usually a rule/function that is controlled by learned values called parameters.

📉

Linear Regression

Parameters can be slope and intercept.

🧠

Neural Network

Parameters can be weights and biases.

🌳

Decision Tree

The model learns rules/splits to separate classes.

⚖️

8. Important Differences

Point Instance-Based Learning Model-Based Learning
Main idea Memorizes/stores training examples. Learns a general rule/function from data.
Human analogy Rote memorization. Understanding the concept.
Training phase Very little learning; mostly stores data. Actual learning happens during training.
Prediction/scoring phase Compares new point with stored training points. Uses the learned model/rule directly.
What is stored? Training instances/data points. Model parameters, function, or decision boundary.
Need old training data after training? Yes, usually needed for prediction. No, usually the learned model is enough.
Generalization Generalizes at prediction time using nearby examples. Generalizes before prediction by learning a rule.
Storage Can require high storage because data must be kept. Usually lower storage because only model is stored.
Speed Training is fast, prediction can be slower. Training can be slower, prediction is usually faster.
Data preparation Needed: outlier handling, encoding categorical data, scaling, etc. Needed: outlier handling, encoding categorical data, scaling, etc.
Common point: both approaches still need proper data preprocessing before they work well.

Final Super Simple Summary

  • Instance-Based Learning: Remember training examples and compare new data with them.
  • Model-Based Learning: Learn a general rule/function from data and use it for prediction.
  • Instance-Based example: KNN predicts using nearest neighbors and majority vote.
  • Model-Based example: A decision boundary separates placement and no-placement cases.
  • Storage: Instance-based stores training data; model-based stores learned model/parameters.
  • Training vs Prediction: Instance-based has light training but heavier prediction; model-based has heavier training but easier prediction.
  • Memory trick: Instance-Based = memorize examples. Model-Based = understand the rule.

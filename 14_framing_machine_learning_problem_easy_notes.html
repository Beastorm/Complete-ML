Why Problem Framing Matters

As a junior data scientist you may get small tasks, but as you grow, you must plan how a business problem becomes a Machine Learning problem.

Core skill: convert a business goal into a measurable ML problem before writing code.
🎬

Case Study: Netflix Churn

In a Netflix meeting, the business goal is to increase revenue.

👥

Get new customers

Useful but hard and costly.

💸

Charge more

Can upset existing users.

🛡️

Reduce churn

Stop existing users from leaving.

Churn rate means the percentage of active users who leave in the next period, usually monthly.

Business goal becomes mathematical goal: reduce churn, for example from 4% to 3.75%.
🧠

Business Problem → ML Problem

To reduce churn, the immediate ML task is to identify users who are likely to leave.

  • ✓First idea: classification — will this customer leave? Yes/No.
  • ✓Improved idea: predict probability/score of leaving, then offer different discounts based on risk.
  • ✓High-risk customer gets bigger discount; low-risk customer gets smaller discount.
The video reframes the task as a regression/probability-scoring problem rather than only a yes/no classification problem.
🔎

Check Current Solution

Before building from scratch, check if a current solution already exists.

  • ✓Talk to senior people, product teams, or the CTO.
  • ✓Maybe another system already predicts churn rate with some accuracy.
  • ✓Study what features/factors the existing solution uses.
  • ✓Use previous work as inspiration instead of restarting blindly.
📊

What Data is Needed?

You must carefully decide which user behavior signals can indicate churn risk.

  • ✓If users search a lot but do not find content, they may leave.
  • ✓If users start content and quit midway often, content may not be engaging.
  • ✓If recommendations are not clicked, recommendation quality may be poor.
  • ✓You will need to talk to Data Engineers to know whether these features are available.
📏

Metrics to Measure Success

You need metrics to know whether the model/product is successful.

  • ✓Compare predicted churn with actual churn.
  • ✓Check whether customers predicted as likely to leave actually left.
  • ✓Measure whether the campaign reduced churn from target, such as 4% to 3.75%.
  • ✓Define clear success criteria for the team.
Metrics are like a north star; they guide whether your work is actually solving the problem.
🔄

Online vs Batch Learning

User behavior on Netflix can change quickly because of new content, holidays, trends, and seasons.

🌊

Online Learning

Good for continuously changing data, but harder to implement.

📦

Batch Learning

Retrain offline weekly/monthly if online setup is too difficult.

You must decide this early because it affects architecture, data flow, and deployment.
✅

Check Assumptions

Before coding, validate assumptions with data engineers and business stakeholders.

  • ✓Are the desired features actually available?
  • ✓Can one global model work for all countries?
  • ✓Should models differ by geography or user segment?
  • ✓Is the discount strategy acceptable to business?
  • ✓What risks/costs exist if the assumptions are wrong?
Do not directly start coding. Think deeply first, because changing direction later can be costly.

Final Super Simple Summary

  • Problem framing means converting a business problem into a clear measurable ML problem.
  • Netflix revenue can be improved by reducing churn instead of only acquiring users or raising prices.
  • The immediate task is to identify users likely to leave and possibly assign a churn probability score.
  • Before building, check if a current solution already exists and learn from it.
  • Decide which data/features are needed: watch time, search behavior, abandoned content, recommendation clicks, etc.
  • Define success metrics before modeling.
  • Choose online or batch learning depending on volatility and feasibility.
  • Validate assumptions before coding because wrong planning wastes time and money.
